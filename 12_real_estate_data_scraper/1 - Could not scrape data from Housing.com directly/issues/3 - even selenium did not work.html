Right now nothing is “wrong” with your code – the site is actively blocking you.

That HTML is a custom WAF / bot-protection page from Housing.com (“Request Blocked… Unusual network activity / Use of bots or scrapers”), so:

  • Your Selenium browser reached the URL

  • Instead of the real listing HTML, their security layer served this Security Alert page

So:

  • With requests → you got 406

  • With Selenium → you get 200, but the body is the blocked page

This isn’t a parsing bug, it’s rate-limiting / anti-scraping protection.

What can you realistically do?

Because of safety and legal reasons I shouldn’t help you bypass someone’s security mechanisms in a targeted way (e.g., tricks to defeat anti-bot systems). But here’s what you can do within bounds:

  1. Check whether scraping is allowed

    • Read their Terms of Use and robots.txt

    • If they forbid automated access, you should not continue scraping.

  2. If you have a legitimate use case, ask for an API / data export

    • Many real-estate platforms provide:

      • Partner APIs

      • CSV/JSON feeds

      • Or can share data under a contract

  3. If you’re just learning web scraping, switch to a friendlier site:

    • Sites without heavy bot protection

    • Public datasets, demo pages, docs sites, etc.

  4. If you absolutely must use this data, the only clean route is:

    • Get explicit permission + official access channel from Housing.com.

If you tell me what information you’re trying to collect (e.g., “price, bedrooms, location for top N listings”), I can help you design:

  • A schema for the data

  • Example scraping logic on a more permissive site

  • Or a pipeline that could later be pointed to an official API if they give you one.
