You are getting 406 from Housing.com because they do content negotiation + bot filtering.

The URL is valid, but requests.get() without headers looks like a bot, so the server refuses to serve your request.

‚úÖ Workaround #1 ‚Äî Add Browser-Like Headers (MOST LIKELY TO WORK)

python
import requests

url = "https://housing.com/in/buy/searches/P1od1w26jrfqap1jlS1Uw5eoY1"

headers = {
    "User-Agent": (
        "Mozilla/5.0 (Windows NT 10.0; Win64; x64) "
        "AppleWebKit/537.36 (KHTML, like Gecko) "
        "Chrome/120.0.0.0 Safari/537.36"
    ),
    "Accept": "text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8",
    "Accept-Language": "en-US,en;q=0.5",
}

response = requests.get(url, headers=headers)

print(response.status_code)
print(response.text[:1000])   # preview

‚úî Pretends to be a normal browser
‚úî Fixes 406 in most cases
‚úî Avoids immediate bot block

‚ö†Ô∏è BUT‚Ä¶ Housing.com also uses Cloudflare bot protection

So even after adding headers, they may still return:

‚ùå 406
‚ùå 403
‚ùå CAPTCHA
‚ùå Empty page

In that case, requests alone will NOT work.

‚úÖ Workaround #2 ‚Äî Use requests_html with JavaScript Rendering

python
from requests_html import HTMLSession

session = HTMLSession()
r = session.get(url)
r.html.render(timeout=20)

print(r.status_code)
print(r.html.text[:1000])

‚úî Renders JavaScript
‚úî Looks more like real browser
‚úî Works better for dynamic housing.com pages

from selenium import webdriver
